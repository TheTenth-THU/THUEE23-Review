## 文件系统概述

操作系统中处理文件的部分称为**文件系统 (file system)**，它管理文件的存储、检索、更新，提供安全可靠的共享和保护手段，并方便用户使用。

文件系统可以从两个角度来理解：

+ **用户观点**：文件如何呈现在用户面前——文件的组成、命名、保护、可进行的操作等。核心目标是**按名存取**
+ **操作系统观点**：文件如何实现——怎样管理存储空间、文件存储位置、磁盘实际运作方式、文件系统性能等

### 文件系统的功能

+ 以符号名称作为文件标识，方便用户使用
+ 支持并发文件访问和控制
+ 在不同设备上提供统一接口（设备无关性）
+ 多种文件访问权限控制
+ 优化性能（存储效率、检索性能、读写性能）
+ 差错恢复——验证文件正确性并具备恢复能力

### 磁盘分区与卷

物理磁盘按**扇区**（通常 512 字节）寻址。**分区 (partition)**是磁盘上一组连续扇区的集合。MBR（主引导记录）位于磁盘 0 号扇区，包含引导代码和分区表。GPT 分区支持更大容量且无主分区数限制。

文件系统在**卷 (volume)**上创建。文件系统格式定义了文件数据在卷上的存储方式。关键术语：

+ **簇 (cluster)**：文件系统可寻址的数据块，大小是扇区大小的整数倍
+ **元数据 (metadata)**：为支持文件系统管理而存储在卷上的数据（如文件和目录的位置信息），对应用程序不可见

## 文件

> [!definition] 文件
> **文件**是一组具有标识符号的在逻辑上具有完整意义的信息项的集合，这个标识符号称为**文件名**。文件包括**文件体**（文件本身的信息）和**文件属性**（存储和管理信息——文件名、内部标识、存储地址、访问权限、访问时间等）。

### 文件的逻辑结构

文件的逻辑结构从用户角度看文件的组织形式：

| 结构类型 | 特点 |
|---------|------|
| **无结构文件**（流式文件） | 文件体为字节流，基本单位是字节。最大灵活性 |
| **累积文件 (pile)** | 无结构记录序列，通过分隔符划分记录，新记录追加到末尾 |
| **顺序文件** | 大小相同的排序记录序列。主文件 + 事务文件，定期归并 |
| **索引顺序文件** | 顺序文件 + 索引 + 溢出文件。索引按关键字区间划分 |
| **索引文件** | 主文件不排序，索引按关键字排序指向记录。支持多索引 |
| **哈希文件**（直接文件） | 记录位置由哈希函数确定，访问速度快但有空间浪费 |

### 文件操作

基本的文件操作包括：创建(`creat`)、打开(`open`)、读(`read`)、写(`write`)、关闭(`close`)、删除(`unlink`)、定位(`lseek`)、获取属性(`stat`)等。

### 文件存取方式

+ **顺序存取**：只能从文件头开始按顺序访问，不能跳过
+ **随机存取**：可以不按顺序访问，或按关键字而非位置来访问

## 目录

**目录**是目录项的有序集合。目录项中包含文件名和查找文件磁盘块所需的信息。文件属性可以存放在目录项中（简单目录），也可以存放在 i 节点中而目录项仅引用 i 节点。

变长文件名的存储方式：
+ **在行中**：目录项包含固定部分和可变部分（文件名），删除后留下可变长度空隙
+ **在堆中**：目录项固定长度，文件名放在目录后的堆中，删除后空隙可被新目录项填充

## 文件的物理结构（实现）

### 连续分配

文件存放在若干连续的簇中。目录项中存放首簇地址和簇数。

+ **优点**：简单易实现；支持顺序和随机存取；顺序存取速度快，寻道次数最少
+ **缺点**：文件不能动态增长；产生外碎片
+ **应用**：CD-ROM 文件系统（文件大小事先已知且不改变）

### 链接分配

文件存放在若干不连续的簇中，前一个簇通过指针指向下一个簇。目录项中只需存放首簇地址。

+ **优点**：充分利用磁盘空间，无外碎片；顺序访问速度快
+ **缺点**：随机访问性能差；寻道次数多；指针占用部分字节导致数据部分不再是 2 的幂次

### FAT 分配

链接分配的改进——将每个簇中的指针字段取出，集中存放在**文件分配表 (FAT)**中。目录中只需记录起始簇号，其余簇从 FAT 表获得。

+ **优点**：消除指针占用数据空间的问题；无外碎片
+ **缺点**：随机访问性能差；FAT 本身占用磁盘空间

### 索引分配

系统为每个文件建立一个专用数据结构——**i 节点**（索引节点），将所有磁盘块号存放在 i 节点中。当文件包含的块数超出 i 节点容量时，最后一项指向包含附加磁盘地址的磁盘块（间接索引）。

+ **优点**：既能顺序存取又能随机存取；满足文件动态增长、插入删除的需求；充分利用外存空间
+ **缺点**：较多的寻道次数和寻道时间；索引表本身带来系统开销

## 磁盘空间管理

### 簇的大小

簇较大时提高 I/O 性能、减小管理开销，但内碎片浪费严重；簇较小时减少内碎片（尤其大量小文件时），但可能导致簇编号空间不足。文件卷容量增大时，要么簇增大（内碎片更多），要么簇编号位数增多。

### 空闲空间管理

+ **空闲块位图**：每个块用 1 位表示（0 空闲 / 1 占用），简单但位图本身占用空间
+ **空闲块链表**：将所有空闲块链接起来，分配和回收操作简单

## 文件共享

### 硬链接

目录项直接指向目标文件的 i 节点。本质是为同一文件创建别名。i 节点维护**链接计数**。

+ **优点**：不占用额外空间，访问速度无损耗
+ **缺点**：不能跨文件系统；不允许指向目录（可能形成死循环）；权限/属性与源文件完全相同

### 符号链接（软链接）

创建一个类型为 LINK 的独立文件，内容为目标文件的路径名。

+ **优点**：可跨文件系统甚至跨网络；有独立 i 节点和权限；允许指向目录
+ **缺点**：源文件删除或改名后断链；访问需二次解析，存在轻微性能损耗；占用额外空间

## 文件系统可靠性

### 文件备份

+ **物理转储**：从磁盘第 0 块开始按顺序输出全部块到磁带。简单，但不能跳过选定目录、不能增量转储
+ **逻辑转储**：从指定目录递归地转储自基准日期以来有更改的文件和目录。需转储通向修改文件/目录的所有目录（即使其本身未修改）

### 文件系统一致性检查

若修改过的磁盘块全部写回前系统崩溃，文件系统可能处于不一致状态。一致性检查包括：

+ **块的一致性检查**：为每个块维护两个计数器——该块在文件中出现的次数 vs 在空闲表中出现的次数。正常状态下每个块两计数器之和应为 1
+ **文件的一致性检查**：检查目录结构和 i 节点引用计数

> [!example]- 常见文件系统格式
> + **FAT 文件系统**：引导区 + FAT 表（两个镜像） + 根目录 + 其他目录和文件。FAT 表中每个簇对应一个表项，文件分配采用链式分配方法
> + **Unix 类文件系统**：超级块（包含文件系统关键参数）+ 空闲块信息 + i 节点数组（每个元素对应一个文件）
> + **NTFS (windows)**：基于主文件表(MFT)，支持日志、权限控制、压缩、加密等高级特性
