## 处理机管理概述

**处理机管理**是操作系统最核心的功能之一，负责协调多个程序对 CPU 的共享使用。多道程序环境下，系统中同时存在多个并发执行的进程，操作系统必须解决四个基本问题：

1. **如何描述进程**——将运行中的程序抽象为进程，通过 PCB 跟踪其状态和资源
2. **如何更轻量地并发**——引入线程，将资源分配与调度执行分离
3. **进程间如何协作**——提供互斥与同步机制，确保共享资源的有序访问
4. **如何分配 CPU 与避免死锁**——选择合适的调度算法，预防和处理死锁

以下依次概述这四个子主题的核心概念与它们之间的关联。

### 进程的描述与控制

**进程**是处理机管理的基本单位。操作系统中，程序的并发执行带来了间断性、失去封闭性和失去可再现性等问题——这正是引入进程概念的动机。

进程的抽象依赖于 **[[进程的描述与控制#进程控制块|进程控制块（PCB）]]**，它是系统感知进程存在的唯一标志。PCB 记录了进程标识、状态、优先级、CPU 现场和资源占用信息。系统将所有 PCB 组织为 PCB 表，其大小决定了系统的**并发度**。

进程在其生命周期中经历多种状态。[[进程的描述与控制#三种基本状态|三态模型]]（运行—就绪—阻塞）是基础，进一步扩展得到[[进程的描述与控制#五状态进程模型|五态模型]]（增加新建态和终止态）和[[进程的描述与控制#七状态进程模型|七态模型]]（增加挂起态）。状态之间的转换由**进程控制原语**（创建、撤销、阻塞、唤醒、挂起、激活）完成——原语是原子操作，不可被中断。

> [!definition] 进程
> **进程**（Process）是具有独立功能的程序在某个数据集合上的一次运行活动，是系统进行**资源分配和调度**的独立单位。进程的组成 = 程序 + 数据 + PCB。

### 线程及其实现

进程同时承担**资源分配单位**和 **CPU 调度单位**两个角色，导致创建、切换进程的开销较大，限制了系统并发度。**[[线程及其实现#引入线程的动机|线程的引入]]**正是为了将这两个角色分离：进程仅作为资源分配单位，线程则成为 CPU 调度的基本单位。

> [!definition] 线程
> **线程**（Thread）是进程中能够独立执行的实体，是**处理器调度的基本单位**。同一进程中的所有线程共享进程的地址空间和资源，但各自拥有独立的 CPU 现场和执行栈。

线程的实现方式决定了系统的并发性能。[[线程及其实现#用户级线程|用户级线程（ULT）]]的线程管理完全在用户空间完成，切换开销极小，但一个线程的阻塞系统调用会导致整个进程阻塞。[[线程及其实现#内核级线程|内核级线程（KLT）]]由内核管理，阻塞在**线程一级**完成，且能在多处理器上并行执行，但模式切换开销较大。[[线程及其实现#混合实现|混合实现]]将 ULT 映射到 KLT 上，兼顾灵活性和性能。

| 方面 | 进程 | 线程 |
|------|------|------|
| 地址空间 | 相互独立 | 同进程内共享 |
| 通信方式 | 需要 IPC 机制 | 可直接读写进程数据段 |
| 切换开销 | 大 | 小 |
| 崩溃影响 | 不影响其他进程 | 可能影响整个进程 |

### 进程间通信

并发进程之间需要两种基本的协调机制。**[[进程间通信#互斥与同步|互斥]]**确保临界资源在某一时刻只能被一个进程访问——这源于无意识的资源共享。**[[进程间通信#互斥与同步|同步]]**则确保协作进程按正确的先后顺序推进——这源于有意识的进程合作。

解决互斥问题的方法经历了从软件到硬件、从分散到集中的演进：

- **[[进程间通信#软件互斥算法|软件方法]]**（锁变量、Peterson 算法）存在忙等待问题
- **[[进程间通信#硬件互斥方法|硬件指令]]**（Test-and-Set、SWAP）利用原子操作简化实现，但仍有忙等待
- **[[进程间通信#信号量|信号量]]**（Semaphore）由 OS 作为管理者，通过 P/V 原语实现无忙等待的互斥与同步
- **[[进程间通信#管程|管程]]**（Monitor）将共享变量及操作封装为模块，由编译器保证互斥

> [!theorem] 临界区访问的四原则
> 1. **空闲让进** 2. **忙则等待** 3. **有限等待** 4. **让权等待**

[[进程间通信#经典 IPC 问题|四个经典 IPC 问题]]（生产者-消费者、读者-写者、哲学家进餐、睡眠理发师）是检验同步机制正确性的标准测试用例。

当进程需要交换大量数据时，可采用[[进程间通信#高级进程间通信|高级通信机制]]：**共享内存**效率最高但需自行处理互斥；**消息传递**通过邮箱/队列实现解耦通信；**管道**以字节流方式在父子进程或任意进程间传输数据。

### 处理机调度与死锁

**[[处理机调度与死锁#处理机调度|处理机调度]]**决定哪个就绪进程获得 CPU。调度分三个层次：长程调度（作业调度，选择作业进入内存）、中程调度（进程换入换出）、短程调度（进程调度，选择就绪进程运行，毫秒级）。

调度算法按应用场景划分：

- **批处理系统**：[[处理机调度与死锁#先来先服务（FCFS）|FCFS]]、[[处理机调度与死锁#最短作业优先（SJF）|SJF]]、[[处理机调度与死锁#最高响应比优先（HRRN）|HRRN]]——关注吞吐量和周转时间
- **交互式系统**：[[处理机调度与死锁#时间片轮转（RR）|时间片轮转（RR）]]、[[处理机调度与死锁#优先级调度|优先级调度]]、[[处理机调度与死锁#多级反馈队列（MLFQ）|多级反馈队列（MLFQ）]]——关注响应时间
- **实时系统**：[[处理机调度与死锁#速率单调调度（RMS）|RMS]]、[[处理机调度与死锁#最早截止时限优先（EDF）|EDF]]——关注截止时间保证

**[[处理机调度与死锁#死锁|死锁]]**是多个进程因竞争资源而无限期等待的僵局。死锁的发生需要四个必要条件同时成立——互斥、非抢占、请求和保持、环路等待。

> [!theorem] 死锁发生的四个必要条件
> 1. **互斥** 2. **非抢占** 3. **请求和保持** 4. **环路等待**

死锁的处理策略由弱到强：[[处理机调度与死锁#死锁预防|死锁预防]]（破坏四个必要条件之一，如预先静态分配法、有序资源使用法）→ [[处理机调度与死锁#死锁避免|死锁避免]]（[[处理机调度与死锁#银行家算法（Dijkstra, 1965）|银行家算法]]，动态判断分配是否使系统进入不安全状态）→ [[处理机调度与死锁#死锁检测与解除|死锁检测与解除]]（通过资源分配图化简检测死锁，剥夺资源或撤销进程解除）。

> [!note] 鸵鸟算法
> 大多数实际系统（UNIX、Linux、Windows）采用"鸵鸟算法"——因死锁发生概率很小而处理代价很大，选择忽略。但这不意味着死锁理论不重要：在数据库、分布式系统等对一致性要求严格的场景中，死锁处理是核心问题。

### 小结

处理机管理的四个子主题构成一条完整的逻辑链：**进程**定义了并发执行的基本单位 → **线程**将资源分配与调度执行分离以提升并发效率 → **进程间通信**提供了并发体之间的协调机制 → **处理机调度与死锁**解决 CPU 的分配策略和资源竞争导致的僵局。这四个环节环环相扣，共同构成了操作系统对 CPU 这一核心资源的管理体系。
