# AGENTS.md

此文件为 opencode 在此仓库中工作时提供项目规范和行文指南。

## 项目概述

清华大学电子系推免复习资料库。将各门课程的知识点整理为 **Obsidian Markdown 文件组**，便于在 Obsidian 中浏览和复习。

## 目录结构

```
<XX> - <课程名>/         # 每门课程一个目录，XX 为课程代码缩写
├── raw/                 # 原始教材 PDF、老师讲义、试题等外部参考资料
├── res/                 # 图片、绘图等附件
├── Index.md             # 课程首页，可包含知识点结构导航的详细说明，**不应保留**课程教师、教材、考试信息等
├── <章节名>.md          # 各章节知识点
├─! CONTENT.md           # **应删除**的目录文件，已由 frontmatter 中的 scenes 定义章节顺序
└─* EXPORT.md            # 可选的导出文件
```

`raw/` 存放该课程的原始资料（教材 PDF、教师讲义、试卷等），可通过 pdf-mcp 工具直接阅读 PDF 文件。

目前已完成 Markdown 化的参考范本：**SS - 信号与系统**、**EM - 电磁场与波**、**SP - 固体物理基础**。

## Markdown 文件格式规范

### 文件组织

+ 每章一个 `.md` 文件，文件名 = 章节标题（如 `Fourier 变换 (FT).md`）
+ `Index.md` 为课程首页，包含：
  + YAML frontmatter（`longform` 配置，列出所有 scene 即章节）
  + 课程信息（可用 `![[]]` 嵌入图片）
  + 知识点导航列表
+ 附件（图片、SVG）放在 `res/` 子目录下，在 frontmatter 中通过 `attachmentFolderPath` 指定

### Frontmatter 格式（Index.md）

```yaml
---
longform:
  format: scenes
  title: <课程名称>
  workflow: Default Workflow
  sceneFolder: /
  scenes:
    - <章节1>
    - <章节2>
    - - <父章节>       # 缩进表示层级
      - <子章节>
  ignoredFiles:
    - CONTENT
    - EXPORT
---
```

### 标题层级

一级标题保留给文件名，章节标题使用二级标题（`##`）开始，子章节依次使用 `###`、`####` 等。

标题前不添加编号，允许使用 LaTeX 宏（如 `\section{\(z\) 变换的定义}` 转为 `## $z$ 变换的定义`）。

### 数学公式

+ 行内公式：`$...$`，不论是否内嵌 `cases` 等环境都必须**保持在同一行内**，公式前后与文本之间至少一个空格，与标点符之间可不留空格
+ 块级公式：`$$...$$`，两个 `$$` 必须**单独成行**
+ 使用 Obsidian 兼容的 LaTeX 语法
+ 自定义宏：`\dif`（微分 d）、`\e`（自然常数 e）、`\I`（虚数 i）、`\v{...}`（向量）、`\dint`（积分）等

### Callout 语法（Obsidian 特有）

```markdown
> [!definition] 标题
> 内容

> [!theorem] 标题
> 内容

> [!note] 标题
> 内容

> [!tip]
> 内容

> [!danger] 标题
> 内容
```

### 内部链接与图片

+ Wikilink 链接：`[[文件名]]` 或 `[[文件名|显示文本]]`
+ 图片嵌入：`![[图片名.png]]`（图片在 `res/` 目录下）
+ 块引用：`[[文件名#^blockid]]`，标注块 ID `^blockid` 应放在引用内容所在行末并间隔一个空格，对 Callout 块放在**标题行**末

### 扩展表格（tx 代码块）

通过 [Table Extended](https://github.com/aidenlx/table-extended) 插件支持扩展的 MultiMarkdown 表格语法，用 ` ```tx ` 代码块包裹。支持以下特性：

**单元格合并**：
+ `^^` 占位符表示该单元格与**上方单元格**合并（纵向合并）。常用于首列的分类标签跨行
+ `||` 在行末表示该单元格**横向合并**至剩余所有列

**行续接**：
+ 行末 `\` 将下一行续接到当前行，使一个逻辑行可跨多个物理行，用于在单元格内嵌入多行内容（列表、代码块等）

**对齐**：
+ 分隔行中 `:---` 左对齐、`:---:` 居中、`---:` 右对齐

示例：

````markdown
```tx
| 信息类别 | 具体内容 | 说明 |
|:---------|:---------|:-----|
| **进程描述信息** | PID | 唯一整数 |
| ^^ | 进程名 | 通常基于可执行文件名 |
| ^^ | 用户标识符 (UID) | |
| **进程控制信息** | 当前状态 | |
| ^^ | 优先级 | |
```
````

其中 `^^` 表示与上一行的第一列合并，使「进程描述信息」跨越 3 行，「进程控制信息」跨越 2 行。

## 行文风格

本节描述知识点 Markdown 文件的行文惯例，参考范本为 **SS - 信号与系统**、**SP - 固体物理基础**（中文为主）和 **EM - 电磁场与波**（英文为主）。

### 段落与节奏

+ 段落简短（1–3 句），公式与文字交替，避免长段纯文字
+ 关键术语首次出现时**加粗**，紧接着给出简短定义
+ 概念引入遵循三步：**粗体术语** → 一句话解释 → 可选的形式化 callout

```
**线性性**是指系统的响应与激励成正比关系。若系统对激励 $e_1(t)$、$e_2(t)$ 的响应分别为 $r_1(t)$、$r_2(t)$，则对激励 $e(t) = a_1 e_1(t) + a_2 e_2(t)$ 的响应为 $r(t) = a_1 r_1(t) + a_2 r_2(t)$。

> [!definition] 线性时不变系统
> **线性时不变（LTI）系统**是同时满足**线性性**和**时不变性**的系统。
```

+ 常用逻辑连接词：`即`（转述公式）、`则`（推出结论）、`其中`（解释符号）、`因此`
+ 非必要不使用括号标注补充内容，而是写入完整句子。例如「程序推进的时序不可再现是允许的，这是并发本身的特征」优于「程序推进的时序不可再现是允许的（并发本身的特征）」
+ 术语附带英文原文时，将原文纳入加粗范围内，使用半角括号、英文小写（专有名词除外）：`**进程 (process)**` 而非 `**进程**（Process）`。缩写全大写保持，如 `**PCB (Process Control Block)**`

### Callout 使用约定

| Callout | 用途 | 示例场景 |
|---------|------|---------|
| `[!definition]` | 形式化定义 | 核心概念的精确定义 |
| `[!theorem]` | 定理、定律、性质 | 卷积定理、Gauss 定律 |
| `[!note]` | 补充说明、概念间关系 | "Laplace 变换与 Fourier 变换的关系" |
| `[!note]+` | 可展开的详细说明 | 较长证明过程 |
| `[!example]` | 典型例题 | 具体计算示例 |
| `[!example]+` | 可展开的长例题 | 需多步推导的示例 |
| `[!tip]` | 实用技巧、记忆口诀 | "微分运算在系统可逆性角度不可逆" |
| `[!danger]` | 常见误区、重要警告 | 易混淆的概念辨析 |

+ 每个章节的 callout 不宜过多（一般 4–8 个），作为聚焦的知识路标
+ `[!note]` 常用于比较和桥接不同概念（如不同变换之间的关系）
+ `[!example]+` 用于较长例题，标题格式 `XXX的应用：示例` 或 `主题—具体题目`
+ `[!note]-` 可折叠，用于非核心的补充推导

### 公式编排

+ 行内公式用于变量引用和简短符号，与周围中文之间留一空格
+ 公式前后常有一句简短引导："满足…"、"定义为…"、"即"、"其中"，尽量不使用冒号
+ 重要公式用 `\mark{...}` 包裹以突出标记
+ 多行推导使用 `\begin{align}`，可嵌套使用 `\begin{aligned}`，关键步用 `\implies` 连接
+ 对较长公式，可在公式后用 `+` 列表逐项解释符号（"where:" 风格）：

```
$$
\v{E}(\v{r};\v{r}') = \dfrac{q}{4\pi\varepsilon_{0}\mathscr{r}^2} \vu{\mathscr{r}}
$$
其中：
+ $q$ 为点电荷电量
+ $\v{r}$ 为场点位置
+ $\v{r}'$ 为源点位置
```

### Wikilink 使用

+ 章节间链接使用 `[[目标章节名]]`，跨课程链接使用完整路径
+ 块链接使用 `[[目标章节#^blockid|显示文本]]`，应始终提供显示文本
+ block ID `^blockid` 放在被引用行的**行末**（同一行，间隔一空格）
+ 对callout的块引用，block ID放在callout **标题行**末
+ 图片嵌入 `![[图片名.png]]`，图片在 `res/` 目录下

### 列表

+ 无序列表使用 `+`，用于枚举性质、分类、步骤
+ 有序列表用于严格的步骤序列（如求解步骤）
+ 子列表应当缩进4空格
+ 列表项中可嵌入公式，公式需额外缩进对齐

### 标点与语言

+ 句子必须带有完整标点，不论是否在列表等环境中，除非结尾「即」「有」「得」紧接行间公式等
+ 使用全角标点 `，` `。` `：` `；` `、`，但内部**只有英文**的括号使用半角括号 `()`，内部含中文时使用全角括号 `（）`
+ 引号仅可使用直角引号 `「」`，不可使用弯引号 `“”` 或直引号 `""`
+ 公式前后与中文之间至少一个空格，而英文与中文之间可不留空格（渲染结果会自动留空），但加粗块另论：
  + 正例：`$\mathcal{F}(\cdot)$ 称为 **Fourier变换**，由Fourier于1822年提出。`
  + 反例：`$\mathcal{F}(\cdot)$称为**Fourier 变换**，由 Fourier 于 1822年 提出。`（必要的空格被省略，导致渲染后词语粘连）
+ 避免口语化感叹词、主观评价、emoji

### 章节结构惯例

每个章节 `.md` 文件通常按以下逻辑组织：
1. 起源/定义/引入背景
2. 基本性质（按子节分列各性质）
3. 典型信号/典型应用
4. 与其他概念的关系（可选）
5. 附录/扩展（可选）

### 章节分层构建流程

对于有父子层级关系的章节，采用自上而下再自下而上的构建方式：

1. **先搭建父章节框架**：在高一级章节文件（如 `Index.md` 或父章节 `.md`）中，列出各子章节的概要，形成完整的知识地图。
2. **仔细撰写子章节**：按照父章节中列出的概要，逐一撰写各子章节 `.md` 文件的详细内容。撰写过程中若发现更好的组织方式或补充内容，可灵活调整，不拘泥于初始概要。
3. **回修父章节**：子章节全部完成后，根据实际撰写的内容修正父章节中的概要描述，确保概要准确反映子章节内容，并补入指向各子章节的 `[[Wikilink]]` 链接。

这一流程确保课程目录结构从始至终保持连贯，各层级之间的概要描述与实际内容一致。

## 工具提示

+ LaTeX 公式在 Markdown 中的正确性需在 Obsidian 中验证，不能仅靠源码检查
+ 各课程的原始 PDF 教材、讲义等存放在 `raw/` 子目录下，可通过 pdf-mcp 工具直接阅读
+ `.tex` 文件中的 tikz 绘图可直接转换为 Markdown `tikz` 代码块
+ 在编辑 `.md` 文件时，避免使用 Obsidian 不支持的 LaTeX 包（如 `nicematrix`、`tcolorbox` 等）
