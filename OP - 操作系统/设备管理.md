## 设备管理概述

I/O 子系统是操作系统中负责管理输入输出设备的部分，其核心设计目标是**设备无关性**（设备独立性）——在设备和系统其他部分之间提供简单且统一的接口，对所有设备都应该相同。

设备管理面临的主要挑战：CPU 与 I/O 设备速度差距悬殊（跨越数个数量级），I/O 性能经常成为系统性能瓶颈；资源多而杂；与其他功能（特别是文件系统）联系密切。

### 设备管理的任务

1. 按照用户请求控制设备操作，完成 I/O 设备与内存间的数据交换
2. 向用户提供方便的外部设备使用接口
3. 充分利用中断、缓冲、DMA 等技术提高 CPU 与设备的并行工作能力
4. 在多进程竞争设备时，按一定策略分配和管理设备
5. 保护数据传输安全
6. 实现**设备无关性**——用户使用逻辑设备名，系统完成逻辑设备到物理设备的转换

## I/O 硬件原理

### I/O 设备的分类

- **按数据组织**：**块设备**（以数据块为单位，如磁盘）vs **字符设备**（以字符为单位，如键盘）；有些设备（如时钟）两者都不是
- **按功能特性**：存储型设备、输入输出型设备（交互型）、数据通信设备
- **按资源分配**：**独占设备**（一段时间内只能一个进程使用，如打印机）vs **共享设备**（多个进程可交叉使用，如磁盘）

### 设备组成

I/O 设备由**机械部分**（物理装置）和**电子部分**（设备控制器/适配器）组成。设备控制器通过标准总线连接主机，通过私有接口连接设备。控制器中设有：

- **控制寄存器**（接收命令）
- **状态寄存器**（报告状态）
- **数据缓冲区**（数据暂存）

控制器接收命令后可独立于 CPU 完成操作，完成后产生中断通知 CPU。

### 端口编址方法

- **I/O 独立编址**：端口地址空间独立于内存地址空间，使用专用 I/O 指令（如 IN/OUT）
- **内存映射编址**：端口与内存统一编址，I/O 操作等同于内存操作（大多数处理器采用）
- **混合方案**：数据缓冲区用内存映射，控制/状态寄存器用独立 I/O 端口（如 PC）

### I/O 控制方式

| 方式 | 特点 | CPU 参与程度 |
|------|------|-------------|
| 程序控制 I/O | CPU 循环查询设备状态（忙等待） | 全程参与 |
| 中断驱动 I/O | 设备完成操作后发中断，CPU 响应处理 | 启动和结束参与 |
| DMA | DMA 控制器接管总线，直接完成内存与设备间数据传送 | 仅启动和结束 |
| 通道方式 | I/O 处理器独立执行通道程序，与 CPU 并行 | 极少参与 |

> [!note] DMA 与中断的关键区别
> - 中断方式：数据缓冲寄存器满后发中断，CPU 控制数据传送
> - DMA 方式：整个数据块传送完成后才产生一次中断，数据传送由 DMA 控制器控制而不经过 CPU
>
> DMA 可通过**周期窃取**（cycle stealing）获取总线周期，大幅减少 CPU 中断处理次数。

## I/O 软件技术

### 缓冲技术

引入缓冲的主要原因：
- 匹配 CPU 与 I/O 设备的速度差异
- 减少对设备的中断频率
- 提高 CPU 与 I/O 设备间的并行性

缓冲类型：单缓冲、双缓冲、循环缓冲（缓冲池）。

### 虚拟设备与 SPOOLing

**SPOOLing**（Simultaneous Peripheral Operations On-Line，假脱机操作）将独占设备改造为可共享的虚拟设备。核心思想：用磁盘作为缓冲，将独占设备的 I/O 操作转化为对磁盘的读写，由守护进程统一管理实际设备。

> [!example]+ SPOOLing 系统组成
> - **输入井/输出井**：磁盘上开辟的缓冲区
> - **输入进程/输出进程**：负责在井和设备间传递数据
> - **井管理程序**：管理井空间和队列
>
> 例如打印机 SPOOLing：多个进程的打印输出先写入磁盘输出井，由打印机守护进程依次将输出井中的文件送往打印机。

### 中断处理

中断处理的一般步骤：
1. 保存被中断进程的 CPU 现场
2. 根据中断向量表转入相应中断服务程序
3. 执行中断服务程序
4. 恢复现场，返回被中断进程（或调度新进程）

> [!note] 流水线与超标量对中断的影响
> 在流水线满的时候，程序计数器可能无法正确反映已执行和未执行指令的边界；在超标量处理器上，乱序执行使这一边界更加模糊。

## I/O 软件层次

I/O 软件通常采用分层结构（由低到高）：

1. **中断处理程序**：响应设备中断
2. **设备驱动程序**：与设备控制器直接交互，每类设备一个驱动
3. **设备无关的 OS 软件**：提供统一接口、缓冲区管理、错误处理、设备分配与回收
4. **用户层 I/O 软件**：库函数（如 `printf`、`scanf`）

## 磁盘设备管理

### 磁盘物理结构

硬盘由多个盘片组成，每个盘面有一个读写磁头。每个盘面划分为同心**磁道**，磁道分为**扇区**（典型 512 字节）。所有盘面上同一磁道构成**柱面**。写入按柱面优先——当前柱面全写满后才移动磁头。

两种寻址方式：
- **CHS**（柱面 Cylinder, 磁头 Head, 扇区 Sector）：三元组定位
- **LBA**（逻辑块寻址）：将 CHS 转换为一维线性地址

### 磁盘 I/O 访问时间

$$
T_{\text{access}} = T_s + T_r + T_t
$$

其中 $T_s$ 为**寻道时间**（磁头移动到指定柱面）、$T_r$ 为**旋转延迟**（等待扇区旋转到磁头下）、$T_t$ 为**数据传输时间**。$T_s$ 通常占主导地位——因此磁盘臂调度算法的目标是减少平均寻道时间。

### 磁盘调度算法

| 算法 | 策略 | 特点 |
|------|------|------|
| FCFS（先来先服务） | 按请求先后顺序 | 公平，性能差 |
| SSTF（最短寻道优先） | 选择距离当前磁头最近的请求 | 减少寻道时间，但可能导致边缘请求饥饿 |
| SCAN（电梯算法） | 在磁头前进方向上选择最近请求 | 无饥饿，但对中间磁道有利 |
| C-SCAN（单向扫描） | 单向移动到边沿后直接折返另一端 | 更公平的等待时间分布 |

### RAID（廉价磁盘冗余阵列）

RAID 通过多个磁盘并行工作，结合条带化（striping）和冗余技术，提高 I/O 性能和可靠性。

| 级别 | 特点 |
|------|------|
| RAID 0 | 条带化，无冗余，读写快但不可靠 |
| RAID 1 | 镜像，容错性好，容量利用率 50% |
| RAID 2 | 字/字节级条带化 + 汉明码纠错 |
| RAID 3-6 | 不同的奇偶校验和条带化策略 |

### SSD（固态硬盘）

SSD 基于 NAND Flash 存储，核心组件为 NAND 闪存、控制器和固件。NAND Flash 的基本单元是浮栅晶体管，写入只能将 1→0，擦除（必须以 Block 为单位）可将 0→1，有擦写次数限制。SSD 维护逻辑地址到物理地址的映射表，省去寻道和旋转时间，读写速度远超 HDD，但成本高、有寿命限制。

> [!example]- 时钟设备
> 时钟（定时器）既不是块设备也不是字符设备。可编程时钟由晶体振荡器、计数器和存储寄存器组成。两种工作模式：一次完成模式（one-shot）和方波模式（周期产生时钟滴答）。时钟软件负责：维护时间、防止进程超时、CPU 使用记帐、处理 alarm 系统调用、提供监视定时器等。
