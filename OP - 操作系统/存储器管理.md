## 存储系统概述

### 存储层次结构

设计一个足够大、足够快又足够便宜的单一存储器是不可能的。现代计算机依靠**存储器访问的局部性原理**构建层次化存储体系：

> [!theorem] 存储器访问的局部性原理
> 处理器访问存储器时，无论取指令还是取数据，所访问的存储单元都趋向于聚集在一个较小的连续区域中。
> - **时间局部性**：最近将要用到的信息很可能就是现在正在使用的信息（主要由循环造成）
> - **空间局部性**：最近将要用到的信息很可能与现在正在使用的信息在空间上邻近（主要由顺序执行和数据聚集存放造成）

存储层次结构（速度递减、容量递增、成本递减）：

| 层次 | 典型介质 | 说明 |
|------|---------|------|
| 寄存器 | CPU 内 | 最快 |
| 高速缓存 Cache | SRAM | Data Cache、Code Cache、TLB |
| 主存储器（内存） | DRAM、SDRAM | OS 存储管理的对象 |
| 辅助存储器（外存） | 磁盘、SSD | 设备管理的对象 |

Cache 对于软件不可见，外存属于设备管理范畴，操作系统存储管理的核心功能针对的是**内存**。

### 存储管理的功能

- **内存分配和回收**：调入策略（请调/预调）、放置策略、置换策略、分配结构
- **地址变换与重定位**：逻辑地址 → 物理地址的映射
- **内存保护**：防止进程越界访问、支持受控共享
- **内存扩充**：覆盖、交换、虚拟存储——逻辑上扩充内存

## 地址变换与程序装入

### 地址空间转换

程序从源代码到可执行文件，经过多次地址转换：

1. **名空间**：程序员使用标识符命名变量、函数——无地址概念
2. **相对地址空间**：编译后目标模块以 0 为基地址顺序编址
3. **逻辑地址空间**（虚拟地址空间）：链接器生成，首地址为 0，其余地址相对首地址编址
4. **物理地址空间**：运行时由机器直接寻址的实际内存地址

**地址重定位**（address relocation）就是将逻辑地址转换为物理地址的过程。

### 程序链接方式

- **静态链接**：事先进行链接形成可执行文件
- **装入时动态链接**：边装入边链接，便于修改、更新和模块共享
- **运行时动态链接**：需要时再链接，节省内存空间

### 程序装入方式

- **绝对装入**：可执行文件中记录实际内存地址，简单但不灵活，不适于多道程序
- **可重定位装入**：一次性实现逻辑地址→物理地址转换，需重定位表，要求程序连续存放且装入后不可移动
- **动态运行时装入**：地址转换推迟到程序真正执行时进行，通过 MMU 硬件完成，是虚拟存储的基础

### 内存保护

通过硬件实现——CPU 中设置**界限寄存器**（基址 + 限长，或上限 + 下限），每次访问主存时硬件自动比较被访问地址是否越界。保护机制在获得许可时应允许受控共享。

## 内存扩充技术

### 覆盖技术

将程序分割成功能上相对独立的程序段（覆盖块），不会同时执行的程序段共享同一块内存区域。覆盖块存放在磁盘上，需要时换入。**由程序员负责分割**，由 OS 完成换入换出。

### 交换技术

当内存空间紧张时，系统将内存中阻塞进程**整体换出**到外存（交换区），把外存中具备运行条件的进程换入内存。这是进程在内存与外存之间的动态调度——对应**中级调度**。

### 虚拟存储器

> [!definition] 虚拟存储器
> 在具有层次结构存储体系的计算机系统中，采用自动实现**部分装入**和**部分交换**功能，为用户提供一个独立于物理内存容量的、可寻址的逻辑存储器。

虚拟存储器以大容量辅存为后备，将程序当前需要使用的部分放在内存、暂时不用的部分放在外存，依赖局部性原理保证正确执行。由 OS 结合 MMU 硬件完成虚拟地址到物理地址的转换。

## 连续分配存储管理

连续分配方式要求为一个用户程序分配一个**连续的**物理内存空间。

### 单一连续区分配

内存分为系统区和用户区，应用程序装入用户区可使用全部空间。适用于单用户、单任务系统（如 MS-DOS）。使用界限寄存器实现保护。

### 固定分区分配

把内存划分为若干个固定大小的连续分区（可等大或不等大）。没有外碎片但可能有内碎片。不等大分区时，放置策略为"将作业放入能容纳它的最小分区"。

- **优点**：易于实现，开销小
- **缺点**：内碎片浪费；分区总数固定，限制并发度

### 动态分区分配

不预先划分分区，当程序需要装入时从空闲内存区中分配大小等于程序所需的空间。没有内碎片但可能产生**外碎片**。

跟踪内存使用情况的两种方法：
- **位图**：内存划分为小的分配单位，每个单位对应一个位（0 空闲 / 1 占用）
- **空闲链表**：链表中每个表项记录占用/空闲标志、起始地址、长度

#### 分区分配算法

| 算法 | 策略 | 特点 |
|------|------|------|
| 首次适配（first-fit） | 从头查找第一个满足要求的分区 | 时间性能好，低端积累小碎片 |
| 下次适配（next-fit） | 从上次分配位置开始查找 | 分布更均匀，大空闲分区不易保留 |
| 最佳适配（best-fit） | 找到大小相差最小的空闲分区 | 保留大空闲分区，但积累小碎片 |
| 最坏适配（worst-fit） | 找到最大的空闲分区 | 不易产生小碎片，但大空闲难保留 |

#### 内存紧缩

当空闲分区总容量大于程序需求但因碎片化无法容纳时，通过移动程序将分散的小分区拼接成大分区——**内存紧缩**（memory compaction）。需动态重定位支持，且十分耗时。

### 伙伴系统

综合固定分区和动态分区的优点。通过不断对分大的空闲分区来获得小的空闲分区，空闲分区的分配和合并都使用 $2^k$ 大小。当内存分区被释放时，检查其"伙伴"是否空闲——若是则合并。

对于大小为 $2^k$、地址为 $x$ 的内存块，其伙伴地址为：

$$
\text{buddy}_k(x) = \begin{cases}
x + 2^k, & \text{if } x \bmod 2^{k+1} = 0 \\
x - 2^k, & \text{if } x \bmod 2^{k+1} = 2^k
\end{cases}
$$

- **优点**：多个空闲链表，查找快速；算法简单
- **缺点**：存在内碎片（总是按 $2^k$ 分配）；只合并伙伴，易产生外碎片；连锁切块/合并可能影响效率

## 非连续分配存储管理——页式

### 简单页式存储管理

将物理内存划分为固定大小的**页框**（page frame），将进程的逻辑地址空间划分为同样大小的**页**（page），以页为单位实现内存分配和回收。逻辑上相邻的页在物理上不一定相邻。

逻辑地址分为两部分：逻辑页号 $P$ 和页内偏移地址 $d$（若页面大小为 $L$ 且为 2 的幂次，则 $P = A / L$，$d = A \bmod L$）。

每个进程有一个**页表**，建立逻辑页号到物理页框号的映射关系。

#### 地址变换过程

1. 从逻辑地址中分离出页号 $P$ 和页内地址 $d$
2. 以 $P$ 为索引查找页表，获得物理页框号 $F$
3. 物理地址 = $F \times$ 页面大小 + $d$

CPU 中的**页表寄存器**（PTR）存放页表在内存的首地址和页表长度。

#### 快表（TLB）

页式存储管理中每次内存访问需两次访存（先查页表，再访问实际数据）。为解决此问题，引入**快表**（TLB，Translation Lookaside Buffer）——一种具有并行查询能力的特殊 Cache，存放当前最常访问的页表项。

- 命中（TLB 中存在该页表项）：取出页框号 + 页内偏移 → 物理地址
- 缺失（TLB 中无此项）：查找页表 → 获得页框号 → 更新 TLB

> [!tip]
> 若一次访存需 1.5μs，无 TLB 时页面访问时间为 3μs。若 TLB 命中率 85%（命中时查找时间忽略），访问时间 = $0.85 \times 1.5 + 0.15 \times 3 = 1.725$ μs。

#### 多级页表

对于大型程序，页表本身可能很大且需要连续存放。解决方法是将页表也分页，形成**多级页表**。例如二级页表结构：逻辑地址分为页目录索引、页表索引、页内偏移地址。

页框大小通常在 512B~4KB 之间，典型大小为 4KB（设置为 2 的幂次以简化硬件逻辑设计）。较小的页面减少内碎片但增大页表，较大的页面提高内外存交换效率。

> [!note] 页式管理的特点
> **优点**：无外碎片（每个内碎片不超过页大小）；程序无需连续存放；便于改变程序占用空间。
> **缺点**：作业一次全部装入内存（不能运行超过物理内存的大作业）；存在内碎片；难于实现真正意义的共享（共享数据可能跨越页边界，与私有数据混合在同一页中）。

### 虚拟页式存储管理

虚拟页式存储管理在简单页式基础上增加**调页**和**页面置换**功能。程序装入时只需将当前需要执行的部分页读入内存即可开始执行。

进程页表需增加：**存在位**（P 位，指示页面是否在内存中）、**修改位**（M 位，指示页面是否被修改过）、访问统计信息、外存地址。

**缺页中断**（page fault）是访问的页面不在内存时由 MMU 产生的中断。缺页中断是特殊的中断——在指令执行期间产生（而不是在指令结束后），一条指令可能产生多次缺页中断。

#### 页面置换算法

当需要调入新页但内存无空闲页框时，必须选择一个页面换出。目标是调出未来不再使用或短期内不使用的页面。

| 算法 | 描述 | 评价 |
|------|------|------|
| OPT（最优） | 置换未来最远才使用的页面 | 不可实现，用作基准测试 |
| NRU（最近未使用） | 根据 R 位和 M 位分为 4 类，随机从最低编号非空类淘汰 | 非常简陋 |
| FIFO（先进先出） | 选择最老的页面置换 | 可能置换重要页，存在 **Belady 异常** |
| 第二次机会 | FIFO + 检查 R 位：若 R=1 则清 R 并移至队尾 | 对 FIFO 的重大改进 |
| Clock（时钟） | 第二次机会的逻辑等价实现（循环链表 + 指针） | 实用 |
| LRU（最近最少使用） | 选择最久未使用的页面 | 性能优异但硬件实现代价大 |
| NFU（最不常用） | 软件模拟 LRU，R 位累加到计数器 | LRU 的近似 |
| 老化（Aging） | NFU 改进：先右移后加 R 位（加权历史） | 有效近似 LRU |
| 工作集 | 维护最近 $\tau$ 时间内的页面集合 | 实现代价较大 |

> [!danger] Belady 异常
> Belady 异常是指 FIFO 算法中，**分配的页框数增多但缺页率反而升高**的异常现象。LRU 等基于访问历史的算法不会出现此异常。

#### 工作集模型

Denning（1968）提出：进程在一段时间内集中访问一些页面（**活跃页面**），如果分配的物理页框数太少导致活跃页面不能全部装入，进程将频繁发生缺页中断——**颠簸/抖动**（thrashing）。

**工作集** $W(t, \tau)$：在时间段 $[t - \tau, t]$ 内进程所访问的页面集合，其中 $\tau$ 为窗口大小。

#### 页面分配与调入策略

- **页面分配**：固定分配（页框数不变）vs 可变分配（根据缺页率动态调整）
- **页面调入**：请调（demand paging，仅调入缺页所需页）vs 预调（prepaging，预先调入可能需要的页）
- **页面清除**：请式清除（替换时写回）vs 预清除（提前写出但仍留内存）vs **页缓冲**（替换页面进入修改/非修改两个队列，周期性成批写出）

> [!note] 简单页式 vs 虚拟页式
> | 特性 | 简单页式 | 虚拟页式 |
> |------|---------|---------|
> | 磁盘使用 | 否 | 物理内存 + 磁盘 |
> | 缺页中断 | 无 | 有 |
> | 页面置换 | 无 | 有 |
> | 内存限制 | 进程 ≤ 物理内存 | 可超过物理内存 |
> | 适用场景 | 嵌入式/实时系统 | 现代通用 OS |

> [!example]- Windows 内存管理
> Windows 11（x86-64）使用四级页表（PML4 → PDPT → PD → PT），每级索引 9 位，页面大小 4KB，支持 2MB 大页和 1GB 超大页。48 位虚拟地址，地址空间分为用户态（低 128TB）和内核态（高 128TB）。安全机制包括 ASLR（地址空间布局随机化）、DEP（数据执行保护）和 KASLR。虚拟地址分配采用两阶段方法（保留 → 提交），通过 VAD（虚拟地址描述符）AVL 树跟踪进程地址空间状态。EXE/DLL 通过内存映射文件加载，利用**写时复制**（copy-on-write）保护共享数据页。页面调度策略：多处理器上采用局部 FIFO，单处理器上采用 LRU。

> [!example]- Linux 内存管理
> Linux 使用基于伙伴系统的物理页框分配器和 slab 分配器（用于内核对象）。虚拟内存支持请调和预调，页面置换采用 LRU 近似算法（活跃/非活跃链表）。支持匿名内存映射（`mmap`）和文件映射，通过四级/五级页表实现地址转换。
